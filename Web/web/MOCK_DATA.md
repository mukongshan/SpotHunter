# 假数据模式说明

当前项目已配置为使用假数据模式，所有API接口都返回模拟数据，无需后端服务即可运行和测试。

## 假数据功能

### 1. 用户系统
- **登录/注册**：输入任意用户名即可登录，如果用户不存在会自动创建
- **用户信息**：每个用户有独立的ID、积分和头像
- **数据持久化**：用户数据存储在内存中（刷新页面会重置）

### 2. 景点数据
预置了5个景点：
- 景区南大门（10积分）
- 中心湖（20积分）
- 观景台（30积分）
- 文化广场（15积分）
- 花园小径（25积分）

### 3. 打卡功能
- 可以正常打卡，获得积分
- 同一天重复打卡同一景点会提示"今天已经打过卡了"
- 打卡记录会保存到内存中

### 4. 数据存储
- 所有数据存储在 `mockData` 对象中
- 刷新页面后数据会重置（因为是内存存储）
- 用户数据、打卡记录都是实时更新的

## 切换到真实API

当后端接口准备好后，可以按以下步骤切换：

1. 打开 `src/api/index.js`
2. 取消注释真实的API调用代码
3. 注释掉或删除假数据相关代码
4. 确保后端服务运行在 `http://localhost:8080`

## 测试建议

1. **测试登录**：输入不同的用户名，观察自动注册功能
2. **测试打卡**：尝试打卡不同景点，观察积分变化
3. **测试重复打卡**：同一天重复打卡同一景点，应该被阻止
4. **测试个人中心**：查看积分和打卡历史
5. **测试后台管理**：查看所有打卡记录

## 注意事项

- 假数据存储在内存中，刷新页面会丢失所有数据
- 图片使用 Unsplash 的占位图，需要网络连接才能显示
- 所有API调用都有模拟的网络延迟（400-800ms）
